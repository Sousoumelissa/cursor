Cahier des charges – Application de gestion des achats restaurant

Client : Restaurateur • Support : Web App Mobile (PWA) connectée à Notion

L'application agit comme interface tactile terrain (cuisine & magasin) tout en gardant Notion comme back-office de référence.

1. Vision & objectifs

Simplicité extrême

UX pensée pour l'usage quotidien, même avec les mains mouillées. Interactions directes et grosses cibles tactiles.

Notion centric

Notion reste le cerveau (catalogue & historique). L'app est la couche de saisie rapide connectée via API.

Mobile first

Interface optimisée pour l'écran vertical d'un smartphone, PWA avec usage hors-ligne et synchronisation différée.

2. Parcours utilisateur

Phase A — Cuisine

  • Affiche le catalogue depuis Notion.
  • Ajout rapide des produits manquants + quantités.
  • Marquage "À acheter" pour alimenter la liste de courses.

Phase B — Magasin

  • Sélection ou modification de la date d'achat.
  • Tri intelligent par rayon et/ou magasin.
  • Cocher, ajuster quantité/prix réel, disparition immédiate.
  • Écriture automatique dans l'historique Notion.

Phase C — Bureau

  • Vue calendrier/liste des sessions d'achats.
  • Total par session et agrégats mensuels.
  • Détail d'une journée avec montants et quantités.

3. Fonctionnalités détaillées

Module 1 — Catalogue

  • Liste déroulante fluide (nom, rayon, magasin, unité, prix standard).
  • Bouton « + » ajout rapide vers la liste d'achat.
  • Recherche instantanée (ex: Avocat).

Module 2 — Shopping list

  • Sélecteur de date visible (« Date de la facture »).
  • Édition live de quantité/prix réel.
  • Tri par rayon ou magasin pour optimiser le parcours.
  • Action « Cocher » qui valide et retire l'article.

Module 3 — Historique & stats

  • Vue calendrier + liste des sessions.
  • Total estimé ou réel par jour.
  • Filtre mensuel et détail par journée.

Module 4 — Configuration & synchro

  • Mode hors-ligne avec file d'attente de synchronisation.
  • Champ de configuration API (Notion/Google Apps Script).
  • Logs de synchronisation (succès/erreurs).

4. UI/UX

Principes

  • Style utilitaire et propre (inspiration Apple/Uber).
  • Typo large, lisible (Inter/Roboto).
  • Palette : fond blanc/gris, bleu action, vert succès, orange alerte.

Ergonomie mobile

  • Actions principales en bas (zone du pouce).
  • Quantités via boutons +/- pour éviter le clavier.
  • Composants tactiles larges, feedback immédiat.
